The purpose of Musify is to provide students-developers with a strong and complete university project. It is not designed to be installed or used yet.
Musify's main code and documentation, including this README file, are preserved in a repository on GitHub shared by the two students-developers of the app.

Configuration

Musify holds some feature that could be customized in the settings section:
  1. Theme: The app can be used in both light and dark mode by clicking the switch next to "Dark" in the settings page
  2. Notifications: Musify Notifications preferences can be updated by clicking on the switch next to "Notifications" in the settings page.
  3. Languages: Musify is mainly programmed in English language and does not support other translations
  4. User Info: Any user informations can be found in the side bar(username - email address - Avatar)
  5. Log in and Log out: The user can log in to the app at any point and is able to sign in by creating his own account.
    In addition, the user can log out of the app by clicking the log out button "settings > log out"
  6. Data Saver: Save your data to enjoy the music offline by turn on the "Data saver" feature in the settings
  7. Video and Audio Quality: Control your Audio and Video preferences in the settings
  8. Password Recovery: In case of lost password the user can request to change his password to regain access to the app using the "forget password?" button in the login page. For Proper Use make sure to write your correct email adrress in it's specified place Before clicking.
  9. Notes: Musify contains some cool animations including one before opening the login page! (could refresh the page using F5 to see the animation 😉)
    In addition this app is designed to be (partially) screen responsive! Make sure to click on the name of the genre to access the playlist!

Developer Guide

Musify is consited of 3 main pages: Login/sign-up,main,settings
Each page has it's own HTML,CSS and JAVASCRIPT files where it is customized carefully for a good user experience

The main page is consisted of 3 sections the main , the music player and the playlist section

VScode is the primary workplace of the students-developers

All HTML,tags used in the base code each work as intended and all codes are written in a semantic format to reflect the neat and professionnal work the students-developers have made

The CSS used in the pages is mainly flexbox format and that helped us create a screen responsive application that suits all types of screens

CSS is also used in some animations and some shadowing especially for the buttons and the loding screen at the pre-login page!

JAVASCRIPT is widely used especially in the settings, it helped us changing the theme for all the pages once applied by the user, as well as in the volume bar to display at what volume the user is pointing at in addition to the save button and the log out button that takes the user to the log in page! As well as in the main page to swtich between the main , the music player and the list of songs

JAVASCRIPT was also used to provide a safe use of the app by adding a firebase thus the user will not be able to operate in the application without signing in first

Git/GitHub was used in the making of this app in order to facilitate the work of the students-developers especially that it helped with the coordination and the easy feature manipulation

Please note that in the production of Musify the students-developers were able to invest there time self-learning in addition to what was taught in class where they were able to use keyframes for animation and variables in CSS to adjust the theme of the application as well as the ability to make a firebase for safe use of the app
